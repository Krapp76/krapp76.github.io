Где же нужен C++?

C++ остаётся незаменимым в нескольких ключевых областях:

  • Большая кодовая база — крупные компании с миллиардными инвестициями в C++ код
  • Экосистема зависимостей — FFmpeg, TensorFlow, PyTorch, математические библиотеки, которые сложно заменить
  • Производительность критична — видеоигры, высокочастотный трейдинг, автопилоты, где время — деньги или безопасность
  • Близость к железу — ядра операционных систем, встраиваемые системы, специфическое оборудование
  • Когда есть только компилятор C++ — для специализированных железок других компиляторов просто нет

"Все браузеры написаны на C++" — Firefox, Chrome, все основные движки.

// Пример высокопроизводительного трейдингового движка
class TradingEngine {
public:
    void process_order(const Order& order) noexcept {
        // Микросекунды имеют значение — прямой доступ к памяти
        auto* slot = order_book_.find(order.symbol);
        if (slot) slot->execute(order);
    }

    OrderBook order_book_;
};

Стандарт C++: 1800-страничный зверь

C++ — один из немногих языков с международным стандартом ISO. Последняя редакция превышает 1800 страниц. Это даёт как мощь, так и сложность:

  • Официальный стандарт стоит ~200 долларов, но бесплатные черновики доступны и почти идентичны
  • Ни один компилятор не реализует стандарт на 100%
  • Предложение фичи занимает от полугода до 3-6 лет
  • Для сравнения: в Elixir можно предложить фичу или исправить баг за недели

"Комитет по стандартизации решает проблемы с помощью неопределённого поведения" — известный мем про C++.

В России можно обратиться к Антону Полухину за помощью с предложениями в стандарт.

// Сложность стандарта: метапрограммирование на шаблонах
template<typename T>
struct is_pointer : std::false_type {};

template<typename T>
struct is_pointer<T*> : std::true_type {};

// Использование
static_assert(is_pointer<int*>::value);     // true
static_assert(!is_pointer<int>::value);    // false

Неопределённое поведение: Тёмное искусство

Неопределённое поведение (UB) — краеугольный камень C++. Оно позволяет компиляторам агрессивно оптимизировать, но возлагает ответственность на программиста.

Ключевые инсайты:

  • UB существует, потому что описать все граничные случаи сделало бы стандарт невозможным
  • "Компилятор не выполняет программу — он компилирует. Результат компиляции вполне определён."
  • Документирование UB в ваших API — легитимный подход для производительного кода
  • Дима Свиридкин пишет отличные заметки о неопределённом поведении — обязательный ресурс к прочтению
// Классическое UB: переполнение знакового целого
int add(int a, int b) {
    // Если a + b превышает INT_MAX, поведение не определено
    return a + b;
}

// Лучше: задокументировать контракт
/// @brief Складывает два целых числа
/// @pre a + b не должно вызывать переполнение
/// @behavior не определено при переполнении
int add_checked(int a, int b) {
    assert(a <= INT_MAX - b);
    return a + b;
}

Память: Где указатели указывают на звёзды

Управление памятью определяет идентичность C++. Знаменитая "звёздочка" — символ указателя — стала иконой:

  • Segmentation fault — обращение к несуществующему адресу, программа падает (это лучше, чем продолжать работу с UB)
  • Современный C++ предлагает умные указатели, но сырые указатели остаются повсеместно
  • Ручное управление памятью даёт производительность, но требует дисциплины

"Раньше учили C++ за 21 день. Теперь предлагают учить за 100 секунд!" — сатира на обещания быстрого обучения.

// Сырые указатели: сила с ответственностью
int* create_array(size_t size) {
    int* arr = new int[size];  // Ручное выделение
    // Не забудьте delete[] потом!
    return arr;
}

// Современный C++: умные указатели
auto safe_array = std::make_unique<int[]>(100);
// Освобождается автоматически

Экосистема C++: Компиляторы, сборка, пакетные менеджеры

Компиляторы: MSVC (Windows), GCC/Clang (Linux/macOS). Операционная система обычно выбирает компилятор за вас — скорее компилятор выбирает программиста, а не наоборот.

Системы сборки: CMake — стандарт де-факто (хоть и сложный). Make, Bazel, Buck тоже существуют.

Пакетные менеджеры: Conan (на Python, гибкий), vcpkg (Microsoft), системные пакетные менеджеры (apt, brew).

Отладчики: Visual Studio Debugger, GDB, LLDB, Xcode.

"Если вы до сих пор не написали свой пакетный менеджер — напишите, там делов-то на два выходных!" (шутка, но с долей правды)

# CMakeLists.txt пример
cmake_minimum_required(VERSION 3.20)
project(MyApp LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

find_package(fmt REQUIRED)
find_package(OpenSSL REQUIRED)

add_executable(my_app main.cpp)
target_link_libraries(my_app fmt::fmt OpenSSL::SSL)

Важные библиотеки C++

Boost — "вторая стандартная библиотека" (используйте осторожно — потом будете рефакторить)

Qt — кросс-платформенный фреймворк для GUI, практически единственный зрелый выбор для сложных приложений

FFmpeg — универсальная молотилка для мультимедиа (видео, аудио, стриминг)

OpenSSL — криптография и SSL/TLS (если видели шифрование на сайте — скорее всего это OpenSSL)

SQLite — встраиваемая база данных, а ещё мощная консольная утилита для анализа данных (местами мощнее Excel)

Google Test — фреймворк для юнит-тестирования

OpenCV — компьютерное зрение (первый шаг в эту область)

TensorFlow / PyTorch — машинное обучение с C++ бэкендами

SDL2 / SFML — создание окон и обработка мультимедиа (клавиатура, мышь, геймпады)

"Awesome C++" на GitHub — отличный репозиторий с библиотеками. Можете добавить туда свою!

// Использование SQLite для анализа данных
#include <sqlite3.h>
#include <iostream>

int main() {
    sqlite3* db;
    sqlite3_open("data.db", &db);
    
    const char* sql = "SELECT name, value FROM metrics ORDER BY value DESC LIMIT 10";
    sqlite3_exec(db, sql, [](void*, int, char** data, char**) -> int {
        std::cout << data[0] << ": " << data[1] << '\n';
        return 0;
    }, nullptr, nullptr);
    
    sqlite3_close(db);
}
